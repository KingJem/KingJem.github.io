phpbrew 使用教程 — 在 Linux/macOS 上管理多个 PHP 版本

什么是 phpbrew

phpbrew 是一个 PHP 版本管理工具，允许你在同一台机器上安装、编译、切换多个 PHP 版本。它从源码编译 PHP，因此你可以自由定制扩展和编译参数，非常适合需要测试不同 PHP 版本兼容性的开发者。

与之类似的还有 phpenv（rbenv 风格）和 phpenv，但 phpbrew 功能最全面，支持的 PHP 版本最广。

指定编译变体（variants）

phpbrew 使用”变体”（variant）机制来组合常用扩展。常用变体：

变体 说明
default 默认常用扩展集（bcmath、curl、json、opcache、pdo、readline…）
mb mbstring + mbregex，多字节字符串支持
dbs sqlite + mysql + pgsql，数据库支持
fpm 编译 PHP-FPM（用于配合 Nginx/Apache）
phpdbg 编译 phpdbg 调试器
opcache Zend OPcache
all 几乎所有扩展（测试兼容性用，编译慢）

常见编译问题处理

缺少库：安装时缺什么库，系统会报错。根据错误信息安装对应 dev 包即可。

Visual C++ 运行库（Windows 下特有）：Linux/macOS 无此问题。

并发冲突：某些扩展组合会冲突，如 mcrypt + sodium。通常去掉冲突扩展即可。

常见问题

Q: 编译时提示 “mbregex” 冲突？
A: 去掉 mbregex 变体，单独使用 mbstring 即可。

Q: macOS 上编译失败？
A: 确保安装 Command Line Tools（xcode-select --install）和 Homebrew 依赖库。

Q: 如何让 phpbrew 使用代理？
A: phpbrew install php-8.2.0 --http-proxy=http://127.0.0.1:7890

Q: phpbrew 自身需要 PHP 多少版本？
A: phpbrew 需要 PHP 7.2+ 作为运行时解释器。
